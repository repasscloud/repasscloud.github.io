About RePass Cloud

Last updated: 2026-02-22

RePass Cloud at a Glance

RePass Cloud Pty Ltd is an Australian software engineering company and consultancy that designs, builds, and operates enterprise-grade software platforms, cloud infrastructure tooling, automation systems, and operational utilities for organisations that need practical, reliable technology built to run in real environments.

We are not a generic software consultancy. We are an engineering-led organisation that builds serious operational software — from large multi-tenant SaaS platforms through to focused tools that solve specific, painful problems across cloud infrastructure, Windows environments, identity, reporting, file systems, and business operations.

Our primary cloud focus is Microsoft Azure and AWS, with selective support for Google Cloud where required. Our delivery spans modern .NET platform development, PowerShell scripting and module development, Windows systems engineering, identity and access automation, and cloud-native design patterns.

Our flagship platform is Cinturon360, a secure multi-tenant enterprise SaaS platform built for organisations that need governed workflows, strong access control, and full operational visibility across complex, layered environments.

What We Do

RePass Cloud operates across five connected capability areas:

1. Enterprise Platform Development

We design and build larger, reusable software platforms intended for commercial deployment at scale. This includes multi-tenant SaaS architecture, identity integration, tenant-aware configuration, audit logging, API-driven integrations, and commercial licensing models. Cinturon360 is the primary example of this capability.

2. Cloud Infrastructure Tooling

We build focused tools for organisations managing cloud environments at scale, including:

  • Cloud cost and spend analysis tooling — visibility into cloud expenditure across Azure and AWS
  • SSL certificate lifecycle management — automated tracking, renewal alerting, and expiration controls
  • Cloud infrastructure automation — provisioning, configuration, and management tooling across Azure and AWS
  • Windows Golden VM automation — automated golden master VM capture for Windows 10 and 11, integrated with Hyper-V and deployable via SCCM or USB

3. Business Operations & IT Tooling

We build practical internal and business-facing tools that remove operational overhead and reduce manual effort, including:

  • ServiceNow ticket management tooling — structured workflow management for IT service operations
  • IT ticket monthly report generator — automated reporting with graphs and trend visualisation
  • Windows machine performance grapher — monthly performance statistics and visualisation for Windows environments
  • Fast delete utility — high-speed bulk file deletion across SMB shares, UNC paths, and direct drive paths, capable of handling hundreds of thousands to millions of files
  • SSH credential manager — structured management of SSH credentials across environments
  • Lightweight SQL DPM tooling — SQL performance monitoring and data extraction utilities
  • Windows Active Directory identity and access automation — structured automation of AD management tasks, identity provisioning, and access lifecycle operations
  • Windows service managers — custom service wrappers and lifecycle management tooling for Windows environments

4. Microsoft 365 & Identity Automation

Many organisations still rely heavily on manual processes to manage their Microsoft 365 and Entra ID environments. We build automation that removes that manual overhead — connecting identity events, HR system changes, and operational triggers to the actions that should happen automatically inside Microsoft 365.

This is a standalone capability, separate from Cinturon360, delivered directly to organisations that need targeted automation around their Microsoft cloud environment.

Common areas we automate include:

  • User onboarding and offboarding — triggered from HR system changes, not manual IT requests
  • User attribute and role changes — licence assignments, group memberships, and access updates driven by position or status changes in the source system
  • Distribution list and shared mailbox management — dynamic, rule-driven membership and access control, including time-based, brand-based, or schedule-driven configurations
  • Identity lifecycle management — ensuring Entra ID and Active Directory remain accurate, current, and clean without manual reconciliation
  • Scheduled and event-driven identity tasks — running identity-related work on a defined schedule or in response to specific triggers, similar to cron-style automation but built for Microsoft 365 and enterprise identity environments
  • Cross-system integrations — connecting Microsoft 365 identity data with HR platforms, finance systems, operational tools, and reporting pipelines

Organisations that have standardised on Microsoft 365 often underestimate how much manual identity and access work their IT teams are carrying. We build the automation that replaces that work — reliably, auditably, and without requiring a large internal development team to maintain it.

5. PowerShell Automation & Scripting

We develop PowerShell scripts and reusable PowerShell modules to automate repetitive operational tasks across Windows, Azure, Active Directory, Microsoft 365, and enterprise IT environments. This includes both standalone scripts and structured modules built to be maintained and distributed across organisations.

Cinturon360

Cinturon360 is our actively developed and maintained enterprise SaaS platform. It is the operational control layer for organisations that need to manage complexity, customers, partners, and internal teams under structured governance without losing visibility or agility.

It is built for organisations that operate in layered environments — where different entities, customers, business units, administrators, and partners all require different levels of access, visibility, and control. Cinturon360 provides a governed foundation for running operational workflows, managing tenant-level configuration, integrating external systems, and maintaining accurate reporting and audit history.

Key capabilities:

  • Multi-tenant architecture with strong isolation and tenant-aware configuration
  • Identity and access management, including Microsoft Entra ID integration
  • Structured operational workflow automation and enforcement
  • Governance controls, role-based permissions, and delegated administration
  • Secure API-driven integration with external identity, payment, and operational systems
  • Full audit logging and operational reporting
  • Regional hosting flexibility to meet data residency requirements

Cinturon360 is not just an automation tool. It is a platform for doing work in a controlled, auditable, and scalable way — one that grows with the organisation rather than becoming a source of technical debt.

Our Approach

We take a pragmatic, engineering-led approach to everything we build:

  • Security-first — identity-centric access, least privilege, strong separation, and auditability baked into design, not bolted on
  • Practical over theoretical — we build software that works reliably in real production environments, not just in demos
  • Tooling-minded — we believe in building focused, maintainable tools that solve specific problems well
  • Automation-heavy — we reduce manual handling, lower process risk, and remove repetitive work wherever it exists
  • Enterprise-capable — our work is designed to run inside organisations with real compliance, access control, audit, and operational requirements
  • Integration-led — we connect with existing infrastructure, identity, payment, and operational systems rather than demanding a full rip-and-replace

Hosting and Regions

We support flexible hosting options to meet customer needs and data residency requirements:

  • Australian and New Zealand customers are hosted in Microsoft Azure Australia regions by default
  • US-based deployments are supported via AWS
  • European deployments are supported via Hetzner where required
  • Additional regions and providers are assessed on a per-customer basis

Technology Focus

Our primary technology focus spans:

  • Microsoft ecosystem — .NET platform development, Azure, Entra ID, Active Directory, Microsoft 365, Windows Server
  • Amazon Web Services — infrastructure, automation, and deployment tooling
  • PowerShell — scripting, module development, and enterprise automation
  • Cloud-native patterns — API design, event-driven systems, observability, and resilient deployment
  • Google Cloud — selective support where required by customers
